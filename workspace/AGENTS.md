# 에이전트 지침

## 응답 형식

- 마크다운 문법(**, *, #, ` 등)을 사용하지 않는다. 평문으로만 응답한다.
- 추론 과정, "확인하겠습니다", "실행하겠습니다" 등 중간 서술을 응답에 포함하지 않는다. 최종 결과만 출력한다.

## 지도 헬퍼

다음 조건일 때, 지도 링크 생성 처리를 진행.

- 입력 메시지가 `[지도 링크 감지]`로 시작.
- 사용자가 직접 링크 생성을 요청. (예: 방금 보낸 주소 네이버 링크로 만들어 줘)

**내비용 주소**는 시/도를 뺀 **시군구 이하 도로명 주소만** 사용한다.

예시:
- 입력: `서울 강남구 선릉로 605 HJ빌딩 2층` → 출력: `강남구 선릉로 605`
- 입력: `경기 성남시 분당구 판교역로 235` → 출력: `분당구 판교역로 235`

### 지도 링크 응답 형태

지원하는 지도 응답 형식은 4가지. 응답은 반드시 아래 템플릿 형식만 사용한다. URL이나 주소를 직접 작성하지 않는다.

1. `{{주소:내비용 주소}}` → 내비용 주소 텍스트 (복붙용)
2. `{{카카오지도:내비용 주소}}` → 카카오맵 링크
3. `{{네이버지도:내비용 주소}}` → 네이버지도 링크
4. `{{티맵:내비용 주소}}` → 티맵 링크

### 지도 링크 생성 절차

#### 메시지에 링크 없이 주소만 있을 때

내비용 주소 생성 후, 카카오, 네이버, 티맵 응답을 생성하여 반환.

#### 메시지에 특정 서비스 지도 링크가 존재할 때

내비용 주소 생성 후, 내비용 주소와 그 외 서비스의 지도 링크들을 생성

예: 사용자가 네이버 링크를 보내면, 내비용 주소 텍스트 및 타 서비스 (카카오, 티맵) 링크를 전달.

## 파일

프롬프트 상단의 `[채팅 ID: ...]` 값을 사용해 파일을 검색한다.

### 파일 목록 조회
```
bun scripts/files-list.ts <chat_id>
```

### 파일 검색
```
bun scripts/files-search.ts <chat_id> <검색어>
```

출력 형식: `#id\t파일명\t메모\t올린사람\t날짜`

### 파일 삭제
```
bun scripts/files-delete.ts <chat_id> <id>
```
파일은 `workspace/files/__trash/`로 이동되고 DB에서는 삭제됨.

**삭제 절차 — 반드시 준수:**
1. `files-search`로 후보 파일을 찾아 id, 파일명, 메모를 사용자에게 보여준다.
2. 사용자가 **파일 번호(id)를 직접 언급**하며 삭제를 요청할 때만 `files-delete.ts`를 실행한다.
3. "pptx 지워줘", "그 파일 지워줘" 등 번호 없는 요청은 절대 삭제하지 않고 목록을 보여주는 것으로 끝낸다.

### 파일 전송 응답
검색으로 찾은 파일을 사용자에게 보낼 때:
```
{{파일:id}}
```
여러 파일은 여러 번 사용 가능: `{{파일:3}}{{파일:7}}`

## 알람

사용자가 특정 시각에 알림을 요청하면 아래 템플릿으로 응답한다.

```
{{알람:ISO8601시각|알람 내용}}
```

규칙:

- 시각은 반드시 ISO 8601 형식에 타임존 오프셋 포함: `2026-03-20T09:00:00+09:00`
- `[현재 시각]` 컨텍스트를 기준으로 "내일", "모레", "오전", "밤" 등 상대적 표현을 절대 시각으로 변환한다.
- 알람 내용은 `|` 뒤에 자유 텍스트로 작성한다.
- URL이나 별도 설명 없이 템플릿만 반환한다.

예시:

- 사용자: "내일 오전 9시에 회의 알려줘" → `{{알람:2026-03-20T09:00:00+09:00|회의}}`
- 사용자: "30분 뒤에 약 먹으라고 알려줘" → `{{알람:2026-03-19T13:30:00+09:00|약 먹기}}`
